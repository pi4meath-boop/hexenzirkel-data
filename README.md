# hexenzirkel-data

Reference data for [Hexenzirkel](https://hexenzirkel.nenri.fr) — the character,
weapon, and artifact-set definitions that drive the damage calculator and
leaderboards.

```
characters/       one <character>.json per playable character
weapons/          one <weapon>.json per weapon
artifact_sets/    one <set>.json per artifact set
```

---

## Conventions that apply everywhere

- **One file per entity.** File names are the lowercase English name with spaces
  replaced by `_` and punctuation removed (`A Thousand Blazing Suns` →
  `a_thousand_blazing_suns.json`). The file name is for humans — the **authoritative
  identity** is the key _inside_ the file (`avatar_id`, `weapon_key`, `set_key`),
  which is what the seeder upserts on.
- **Stat-value units.** Percentage stats — every key ending in `_`
  (`atk_`, `critRate_`, `critDMG_`, `enerRech_`, `pyro_dmg_`, `all_dmg_`, `heal_`, …)
  — are written as **ratios**: `0.15` means **+15 %**. Flat stats
  (`atk`, `hp`, `def`, `eleMas`) are written as **raw numbers**: `1204.0`.
- **Fixed assumptions.** Generated numbers are captured at **character level 90**,
  **all talent levels 10**, **constellation 0**, and **weapons at refinement 1
  (R1)**. Constellation- and refinement-dependent effects are expressed separately
  (see _gating_ below), not baked into the base numbers.
- **Elements**: `pyro hydro electro cryo anemo geo dendro physical`.
- **Stat keys** must match the ones the damage evaluator understands. The simplest
  way to stay correct is to **reuse keys that already appear in existing files**
  rather than inventing new ones.
- Every file is plain JSON — it must parse, and 2-space indentation is preferred.

## Where the numbers come from

Most of the numeric data is generated by internal tooling from public datasets,
then the situational buffs are authored by hand:

| Part                                           | Source                                                                                                                                                                                                               |
| ---------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Talent multipliers (`talent_multipliers_json`) | [GenshinOptimizer](https://github.com/frzyc/genshin-optimizer) `allStat_gen.json`, read at talent level 10. The `source` block inside the file records the exact `goCommit`, `goCharacterKey`, and assumptions used. |
| Base stats (`base_stats_json`)                 | [Ambr.top / Project Amber](https://ambr.top) — level-90 base HP/ATK/DEF plus the ascension stat (`ascStatKey` / `ascStatValue`).                                                                                     |
| Localized names                                | Project Amber (handled by the app, not stored here).                                                                                                                                                                 |
| Buffs & conditional modifiers                  | **Hand-authored** from the in-game talent / passive / artifact descriptions.                                                                                                                                         |

When you contribute generated values, cite where they came from (a GO commit, an
Amber page, or the in-game text) in the pull-request description.

## File schemas

### Character (`characters/*.json`)

```jsonc
{
  "display_name": "Bennett",
  "go_key": "Bennett",                 // GenshinOptimizer key
  "avatar_id": 10000032,               // authoritative identity (upsert key)
  "character_key": "avatar_10000032",  // defaults to "avatar_<avatar_id>"
  "element": "pyro",
  "weapon_type": "sword",
  "nation": "Mondstadt",               // optional
  "constellation_talent_boosts": { "skill": 3, "burst": 5 },
  // ^ which constellation grants that talent +3 levels (C3 → skill, C5 → burst)

  "base_stats_json": {
    "baseHp": 12397.4, "baseAtk": 191.16, "baseDef": 771.25,
    "ascStatKey": "enerRech_", "ascStatValue": 0.2667
  },

  // What each hit is: element, which DMG-bonus stats apply, and how it scales.
  "hit_definitions_json": {
    "normal": {
      "hit1Dmg": {
        "element": "physical",
        "dmgBonusKeys": ["normal_dmg_", "physical_dmg_", "all_dmg_"],
        "components": [ { "scalingKey": "atk", "multKey": "p0_mult" } ]
        // damage of a component = <scalingKey stat> × <multKey value below>
      }
    },
    "skill":  { "skillDmg": { … } },
    "burst":  { "hit1":     { … } }
  },

  // The multiplier each multKey refers to, per talent, at talent level 10.
  "talent_multipliers_json": {
    "normal": { "p0_mult": 0.8806, "p1_mult": 0.8449, … },
    "skill":  { "p0_mult": 2.4768, … },
    "burst":  { "p0_mult": 4.1904, … },
    "passive": {}, "constellation": {},
    "source": { "goCommit": "…", "goCharacterKey": "Bennett", "assumptions": { … } }
  },

  "buffs": {
    "self_modifiers_json": {},              // always-on buffs to the wearer
    "self_conditional_modifiers_json": [],  // conditional buffs to the wearer
    "team_modifiers_json": {},              // always-on buffs to the team
    "team_conditional_modifiers_json": [    // conditional buffs to the team
      {
        "conditionKey": "burstInspirationField",
        "source": "Bennett",
        "states": [ { "value": "on", "modifiers": { "atk": 1204.0 } } ]
      },
      {
        "conditionKey": "c6PyroInfusionField",
        "source": "Bennett",
        "gating": { "constellationMin": 6 },   // only exists at C6+
        "states": [ { "value": "on", "modifiers": { "pyro_dmg_": 0.15 } } ]
      }
    ]
  }
}
```

### Weapon (`weapons/*.json`)

```jsonc
{
  "weapon_key": "12514", // authoritative identity (upsert key)
  "display_name": "A Thousand Blazing Suns",
  "rarity": 5, // optional
  "base_stats_json": {
    "baseAtk": 740.58,
    "subStatKey": "critRate_",
    "subStatValue": 0.110256,
  },
  "buffs": {
    "self_modifiers_json": {},
    "self_conditional_modifiers_json": [
      {
        "conditionKey": "sunset_reignites_dawn",
        "description": "After skill/burst: CRIT DMG +20%, ATK +28% for 6s.",
        // states can also be a map of named states → modifiers:
        "states": {
          "passive_only": { "critDMG_": 0.2, "atk_": 0.28 },
          "nightsoul_blessing": { "critDMG_": 0.35, "atk_": 0.49 },
        },
      },
    ],
    "team_modifiers_json": {},
    "team_conditional_modifiers_json": [],
  },
}
```

Weapon buffs are written at **R1** for 5* weapons and **R5** for 4* weapons.

### Artifact set (`artifact_sets/*.json`)

```jsonc
{
  "set_key": "ArchaicPetra", // authoritative identity (upsert key)
  "display_name": "Archaic Petra",
  "buffs": {
    "self_modifiers_json": { "geo_dmg_": 0.15 }, // 2-pc: always on
    "self_conditional_modifiers_json": [],
    "team_modifiers_json": {},
    "team_conditional_modifiers_json": [
      // 4-pc, situational
      {
        "conditionKey": "4pc_upon_obtaining_an_elemental_shard",
        "source": "ArchaicPetra",
        "description": "4pc: all party members gain a 35% DMG Bonus…",
        "modifiers": {
          "pyro_dmg_": 0.35,
          "hydro_dmg_": 0.35,
          "electro_dmg_": 0.35,
          "cryo_dmg_": 0.35,
        },
      },
    ],
  },
}
```

### Modifier shapes (buffs)

A **flat modifier map** (`*_modifiers_json`) is just `{ statKey: value }` and is
always applied. A **conditional modifier** (`*_conditional_modifiers_json`) is an
array of objects; each has a `conditionKey`, an optional `description`, `source`,
and `gating` (e.g. `{ "constellationMin": 6 }`), and carries its values in one of
these forms — **match whichever an existing similar entry uses**:

- `"modifiers": { … }` — a single set of values (see Archaic Petra 4-pc).
- `"states": [ { "value": "on", "modifiers": { … } }, … ]` — discrete toggle states.
- `"states": { "stateName": { … }, … }` — named stacking/level states (see the weapon).

## Contributing (pull / merge requests)

Changes land through **GitHub pull requests** against
[`Nenrikido/hexenzirkel-data`](https://github.com/Nenrikido/hexenzirkel-data).

1. **Fork** the repo and create a branch:
   `git checkout -b add-<character-or-weapon-or-set>`.
2. **Add or edit** the relevant JSON file, following the schema, naming, and
   value conventions above. Copying the closest existing file and adapting it is
   the fastest way to get the shape right.
3. **Validate the JSON** — it must parse. Quick check:
   ```bash
   python -c "import json,sys; json.load(open(sys.argv[1]))" characters/<file>.json
   ```
4. **Commit** with a clear message and **open a pull request**. In the description,
   say where the numbers came from (GO commit / Amber page / in-game text) so a
   maintainer can verify them.

### Checklist for a new entry

- [ ] File named after the lowercase English name (`_` for spaces, punctuation removed).
- [ ] Authoritative key set and unique (`avatar_id` / `weapon_key` / `set_key`).
- [ ] Percentages as ratios (`0.15`), flats as raw numbers.
- [ ] Stat/element keys reuse ones already present in other files.
- [ ] Generated numbers captured at L90 / talents 10 / C0 / 5* weap at R1, 4* weap at R5, with sources cited.
- [ ] JSON parses.
